# Teste de habilidades Software Developer - Backend

## Informações gerais:

Não publique seu código no GitHub em repositório público. Envie o código em uma pasta em formato zip.

## Características pelas quais você será avaliado - por ordem de importância:

1. Qualidade geral do código;
1. Uso de testes;
1. Qualidade do README;
1. Componentização / Acoplamento / Divisão de responsabilidades;
1. Facilidade de execução;
1. Uso de padrões de projeto;
1. Design do código;
1. Funcionamento do código.

## O seu arquivo README.md deve conter:

1. Descrição e explicação do seu projeto (até 300 palavras)
  1. Como ele funciona?
  1. Porque você escolheu a estrutura que aplicou?
1. Como executar seu código?
1. Como fazer a chamada dos endpoints criados?
1. Quais seriam seus próximos passos para transformar este código em um microserviço pronto para ir ao ambiente produtivo?
1. Quaisquer outras informações que você queira enviar.

## Avaliação:

Desenvolva uma aplicação para controlar um cadastro de bicicletas. Sua aplicação deve ter as seguintes operações HTTP:

1. GET – listar todas as bikes cadastradas;
1. GET – listar uma bike por ID;
1. DELETE – deletar uma bike;
1. PUT – atualizar todas as informações da bike;
1. POST – criar uma bike;
1. PATCH – atualizar alguma informação específica de uma bike;

Sugestão de campos para a tabela de bikes: descrição, modelo, preço, data de compra, nome do comprador, nome da loja em que comprou.

## Para os candidatos JAVA:

O objetivo final é que você crie uma aplicação com Spring e as características abaixo:

1. Usar Spring boot;
1. Usar banco MySQL, PostgreSQL ou H2;
1. Usar JPA;
1. Adicionar testes automatizados;
1. Usar as features do Java 8/11 de programação funcional apropriadamente;

Opcional (bônus): Disponibilizar a versão final em uma nuvem pública (como Azure, AWS, Heroku, DigitalOcean, ou GCP);

## Para os candidatos PHP:

O objetivo final é que você crie uma aplicação com o framework PHP de sua preferência e as características abaixo:

1. Usar banco MySQL ou PostgreSQL ou H2;
1. Adicionar testes unitários;
1. Usar as features das versões mais atuais do PHP;

Opcional (bônus): Disponibilizar a versão final em uma nuvem pública (como Azure, AWS, Heroku, DigitalOcean, ou GCP).
